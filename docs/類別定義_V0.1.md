## C-1 商品內容（content）

> 建議動作：`clarify_wording`

**定義**：【商品內容（C-1）】評論抱怨商品頁『描述本身』不清楚/模糊/籠統/沒寫/誇大不實/誤導——判準是『改頁面文案就能解決』。非實際體驗差(→C-2/C-3)、非兌換卡關(→C-4)、非客服(→C-5)、非主觀期待或頁面有寫沒看(→C-6)。

**✅ 屬本域**

- 頁面只寫『半天』沒寫具體幾小時，行程總時長交代不清
- 頁面完全沒提要另付門票/費用，或費用項目寫不清楚
- 頁面沒清楚標示集合等候地點，或資料本身給錯地點
- 頁面介紹很籠統含糊，看不懂實際要玩什麼內容
- 頁面沒寫清楚兌換/使用方式導致認知落差（非人員刁難）
- 頁面沒寫清楚退改政策細節（非客服互動品質問題）

**❌ 常見誤判**

- 行程頁面寫X小時，實際到場縮水/提早解散→歸供應商履約，非商品內容
- 頁面已具體寫明含某項目，現場卻要另外自費/沒提供→歸供應商履約，非商品內容-費用
- 頁面已寫清楚集合時間地點，客人自己沒看/看錯→歸客人理解，非商品內容
- 跟照片有落差、覺得不如預期、覺得不值→歸客人理解-期待落差，非商品內容
- 頁面寫得清楚，但客人詢問時客服應對慢或答非所問→歸客服營運，非商品內容

**⛔ 明確禁止**

- 頁面已具體寫明數值/項目時，禁止因『執行縮水』改判商品內容
- 客服互動品質問題（回應慢/態度差）禁止歸商品內容

| C-1-2 | 行程流程 | 行程流程資訊寫錯或缺漏：評論『明確指出頁面/介紹』沒寫清或與實際不符（時長/步驟/景點清單/交通/自費項目）；核心是「評論有提到頁面資訊本身的問題」。判定預設：評論若『沒有』提及頁面/介紹/說明怎麼寫，僅抱怨現場時間安排或行程內容 → 禁歸本類（依表嫌趕嫌短→C-6-3、執行偏離表定→C-3-3、現場才知要另付費→C-1-3 僅限費用類）。 |

## C-2 商品品質（quality）

> 建議動作：`penalize_breach`

**定義**：【商品品質（C-2）】評論抱怨產品/服務『交付物本身』的客觀品質差——吃到/住到/坐到/連到的東西不行（網路/餐飲/車輛/住宿/設施）。非人為執行差→C-3、非頁面與實際不符→C-1、非主觀不值→C-6-2

**✅ 屬本域**

- 抱怨網路訊號差/連線不穩定，非設定操作問題本身
- 抱怨餐點難吃/份量少/食材差，屬實際餐飲品質
- 抱怨車輛老舊/座椅不適/設施骯髒等實體品質
- 抱怨住宿設施故障、隔音差等實體體驗品質
- 因天候封艙加蓋導致悶熱不通風、船帆/設備無法升起使用——起因雖是天候，設施體驗品質差仍歸本域（可與不可抗力並列雙歸因）

**❌ 常見誤判**

- eSIM/門票無法連線啟用或核銷失敗（操作/系統問題）→ 歸平台與系統，非商品品質
- 頁面完全沒寫清楚會提供什麼等級的餐飲/設施 → 歸商品內容，非商品品質
- 供應商承諾包含某項目卻沒提供（該給沒給）→ 歸供應商履約，非商品品質
- 行程步調太趕/沒看到景點，屬安排節奏問題 → 歸供應商履約，非商品品質
- 單純覺得餐點不合個人口味、非品質瑕疵 → 歸客人理解，非商品品質

**⛔ 明確禁止**

- 供應商人員執行不到位或服務態度差（非交付物本身品質問題）→ 歸 C-3
- 頁面描述與實際交付內容不符（貨不對板，非物理品質本身差）→ 歸 C-1
- 連線/核銷/開通等操作面失敗問題，非交付物實體品質差 → 應歸平台與系統（C-4）
- 單純主觀覺得不值/不合期待、交付物本身無瑕疵 → 歸 C-6-2/C-6-3

**L2 面向**

| code | 面向 | 定義 |
|---|---|---|
| C-2-1 | 網路品質 | 抱怨行動網路/eSIM 訊號微弱、網速緩慢或完全斷線等連線品質本身差；核心是「訊號/速度/連線本身」。非開通啟用操作卡關→C-4-1、非客服申訴處理消極→C-5-3 |
| C-2-2 | 餐飲品質 | 餐點難吃、味道差（過鹹過淡/微溫）、食材不新鮮或選項太少，或用餐座位擁擠、現場衛生差（如有蒼蠅）；核心是「吃得好不好、選得夠不夠、現場乾不乾淨」。非該給沒給→C-3-4、非不合口味→C-6-3 |
| C-2-3 | 車輛設備 | 付了較高等級車型的錢現場卻派低等級車輛，或車輛/設備本身狀態不佳、有明顯故障；核心是「車不符約定或車況不行」。駕駛行為→C-3-2、安全配備缺損→C-3-5 |
| C-2-4 | 住宿品質 | 住宿房間本身實體品質差——髒亂、有異味/蟲、設備故障（冷氣/熱水/Wi-Fi 壞）、隔音差、備品或清潔不到位；核心是「住宿實體品質不佳」 |
| C-2-5 | 設施設備 | 參加活動或使用場館時，現場器材/遊具/場地設施本身狀態不佳——老舊/故障/不潔、遊樂或展館設施損壞、場地維護不良；核心是「活動/場館的實體設備品質」 |

## C-3 供應商履約（supplier）

> 建議動作：`penalize_breach`

**定義**：【供應商履約（C-3）】評論抱怨供應商『現場的人與執行差』——到場後的服務、駕駛、節奏、履約、安全管理沒依約做到（縮水/遲到/擅改/態度/風險；頁面已揭露）。非東西本身差(→C-2)、非系統流程卡關(→C-4)、非售後客服(→C-5-3)、非頁面本身模糊沒寫(→C-1)、非外力事件本身且已合理應變(→C-6-4/C-6-5)、非客人自身外力因素(→C-6)。

**✅ 屬本域**

- 頁面已寫明具體行程內容/時數，現場卻縮水、提早結束
- 頁面已寫明包含某項目（如餐點/門票），供應商現場卻不提供或要求加價
- 供應商遲到、擅自更改行程安排或路線
- 現場人員服務態度差、帶團節奏過趕
- 頁面已具體告知集合資訊，供應商卻臨時更改地點未通知
- 現場安全配備缺損或衛生條件不佳，讓人擔心健康與人身安全風險
- 現場施壓強迫消費、巧立名目強收費用或歧視差別待遇

**❌ 常見誤判**

- 頁面本身沒寫清楚時數/內容，客人才誤會→歸商品內容，非供應商履約
- 抱怨實體品質本身差（餐難吃/車舊），非供應商『沒做到』→歸商品品質，非供應商履約
- 兌換/核銷當下操作卡關（連線/入場失敗）→ 歸 C-4-2（憑證與資格），非供應商履約
- 客人自身理解錯誤或沒看須知，商家其實有做到→歸客人理解，非供應商履約
- 因天災/外力取消且供應商已合理應變→歸客人理解（C-6-4/C-6-5）；明知風險未告知或應變失當→仍屬供應商履約（C-3-6）

**⛔ 明確禁止**

- 頁面本身模糊不清時禁止歸供應商履約（應先判是否商品內容問題）→ 歸 C-1
- 純外力事實且供應商已合理應變 → 歸 C-6-4/C-6-5；明知風險仍未告知、無備案或應變失當 → 歸 C-3-6，仍屬本域
- 客人自身外力因素（航班延誤/臨時無法出席）非供應商過失 → 歸 C-6
- 系統/平台流程本身卡關（連線/核銷/帳號）非供應商執行問題 → 歸 C-4
- 事後客服應對品質問題（非現場執行）→ 歸 C-5-3

**L2 面向**

| code | 面向 | 定義 |
|---|---|---|
| C-3-1 | 人員服務 | 抱怨現場人員（導遊/司機/接待/店員）態度、語言溝通、講解專業能力不足、或對特殊需求協助不足，非售後客服（→C-5-3）。 |
| C-3-2 | 駕駛接送 | 抱怨接送遲到/找不到/危險駕駛，非車況本身差→C-2-3 |
| C-3-3 | 帶團節奏 | 現場執行『偏離表定』的節奏調度問題：司機/行程遲到等人集合過久、臨時壓縮或取消停留點、比表定提早收團、被加插購物站耽誤行程；核心是「評論明確寫出實際執行與行程表不一致的事實」。判定預設：評論若『沒有』寫出偏離表定的具體事實，僅嫌趕、嫌停留短、嫌拉車遠、嫌購物站太多（頁面已載）→ 一律歸 C-6-3，禁止因『時間/節奏』字眼歸本類。頁面根本沒寫清時長流程→C-1-2、塞車天候等外力→C-6-4/C-6-5、接送環節本身遲到→C-3-2。 |
| C-3-4 | 約定履行 | 供應商執行面未落實原訂約定：人數不足未成團或臨時取消且未主動及時告知、現場擅改餐點/行程未告知、方案載明應提供的票券/贈品/項目現場卻沒給、憑證未如期發出、現場臨時追加收費未事先告知。已知風險/不可抗力情境下的應變與告知失職 → 歸 C-3-6。 |
| C-3-5 | 現場安全與衛生 | 供應商現場存在人身安全風險或健康疑慮：安全配備缺損（安全帶/救生衣/頭盔等）、車輛或場地超載擁擠、人流控管與動線規劃失當致擁擠推擠風險、急救資源缺失、現場管理不當致人身風險、衛生條件差到足以造成健康疑慮，聚焦「風險與健康疑慮」而非單純不舒適。 |
| C-3-6 | 風險應變與告知 | 供應商對已知風險的應變與告知失職：明知惡劣天候/罷工/場館關閉等風險將至，仍未主動告知、未提供備案、未合理取消或應變處置失當（強行出團/直接判全損/丟包），以及發生事故或不適後的善後關懷缺失；核心是「商家對事件的『處置』有明確失職事實」，外力事件可不可控不是重點。鏡像判準：外力事件本身（無失職事實）→ C-6-4/C-6-5，可並存；非風險情境的一般約定未履行 → C-3-4。 |
| C-3-7 | 不當行為 | 供應商現場的惡意/失格行為（超出服務品質範疇、涉商業誠信與人格對待）：話術施壓或變相強迫購物消費、強迫升級付費項目、巧立名目強收費用、以沒收訂金/不退款要脅，以及因國籍/語言/身分的歧視與差別待遇、操縱評價（造假好評、以贈品/優惠利誘換好評、施壓或脅迫要求好評）；核心是「行為的惡性程度」，非單純態度不佳（→C-3-1）、非購物站時間侵占（→C-3-3）。屬高嚴重度，應觸發供應商管理/法務關注。 |

## C-4 平台與系統（platform）

> 建議動作：`escalate_ops`

**定義**：【平台與系統（C-4）】平台與系統：人沒錯、東西沒壞——『下單→開通→兌換→使用』的系統流程卡關：開通啟用、憑證與資格、平台功能。非頁面沒寫清→C-1-5、非現場人員刁難→C-3-1、非售後求償→C-5

**✅ 屬本域**

- 開通/啟用環節卡關，商品開不起來
- 核銷/兌換當下操作卡關、無法完成入場動作
- 系統誤判憑證已使用、導致實際未使用卻無法申請
- 預約/資格規則把人卡住，約不到或被規則擋下
- App/網站功能異常，下單或查詢流程卡關

**❌ 常見誤判**

- 頁面沒說明開通/兌換方式導致客人不會操作→歸商品內容，非平台與系統
- 現場人員刁難、拒絕受理兌換（人員態度問題）→歸供應商履約，非平台與系統
- 兌到的票券/商品本身品質差（非流程卡關）→歸商品品質，非平台與系統
- 退款/改期申請流程與客服往來問題→歸客服營運，非平台與系統
- 行李/尺寸等規範資訊不足，客服未能回答→歸客服營運，非平台與系統（非流程卡關本身）

**⛔ 明確禁止**

- 頁面沒寫清開通/兌換方式，禁止歸平台與系統（應歸商品內容 C-1-5）
- 現場人員刁難、態度不佳，禁止歸平台與系統（應歸供應商履約 C-3-1）
- 退款/改期等售後求償問題，禁止歸平台與系統（應歸客服營運 C-5）

**L2 面向**

| code | 面向 | 定義 |
|---|---|---|
| C-4-1 | 開通啟用 | 商品開通/啟用環節卡關，或效期起算/提前啟用爭議；核心是「開不起來或一開通就吃虧」。開通後品質差→C-2-1、頁面沒教怎麼用→C-1-5 |
| C-4-2 | 憑證與資格 | 憑證核銷/取票/入場/出示環節失敗，或預約/資格/份數規則把人卡住；核心是「換不了、被規則卡住」 |
| C-4-3 | 平台功能 | 平台自身功能壞了；核心是「平台壞了」 |

## C-5 客服營運（service）

> 建議動作：`escalate_ops`

**定義**：【客服營運（C-5）】評論抱怨『售後客服互動』的人與政策處理差(退款/回應/改期/態度)；核心是「售後的人與政策」。非現場人員→C-3-1、非平台系統故障→C-4-3、非條款未揭露→C-1-7

**✅ 屬本域**

- 客服應對緩慢、答非所問、反覆要求提供證明卻無進展
- 退款/改期申請流程本身處理不當或標準不一致
- 客人已提供證明資料，客服仍未妥善處理或拖延
- 客服轉真人客服後久候無回應，或處理過程態度不佳

**❌ 常見誤判**

- 客人單純詢問取消政策/流程，客服正常依規則回答（非互動品質問題）→歸客人理解，非客服營運
- 頁面本身沒寫清楚退改規則→歸商品內容，非客服營運
- 現場人員（供應商端）態度差或刁難→歸供應商履約，非客服營運
- 兌換系統本身操作卡關、非人工客服互動問題→歸平台功能，非客服營運
- 因天災不可抗力申請退款，客服依規則正常說明條件→歸客人理解，非客服營運

**⛔ 明確禁止**

- 客人單純提問、客服正常依政策回覆，禁止僅因『涉及退款/客服』就歸客服營運
- 天災導致的取消/改期本身禁止歸客服營運

**L2 面向**

| code | 面向 | 定義 |
|---|---|---|
| C-5-1 | 確認/修改 | 抱怨改期/確認沒落實或退改政策設計不清；核心是「客服銜接與條款設計」。頁面沒寫政策→C-1-7；系統功能本身壞→C-4-3 |
| C-5-2 | 取消/退款 | 抱怨退款被拒或延宕(錯不在客人)，非頁面沒寫退款規則→C-1-7 |
| C-5-3 | 客服應對 | 抱怨平台/線上客服的回應時效與立場態度(已讀不回/答非所問/推諉/繁瑣)；被稱『客服』的當地窗口/導遊仍屬現場→C-3-1 |

## C-6 理解期待（customer）

> 建議動作：`escalate_ux`

**定義**：【理解期待（C-6）】旅客端與外力：商品與執行皆正常，問題在主觀認知、自身因素或外力事件；有具體瑕疵事實→C-2/C-3。非頁面確實模糊沒寫→C-1、非實際執行縮水→C-3、非實體品質差→C-2、供應商對已知風險的應變與告知失職→C-3-6。

**✅ 屬本域**

- 客人自身理解錯誤、沒看清楚須知或憑證規則
- 客人主觀覺得不如預期、不值、跟照片有落差但頁面已寫清楚
- 因天災/不可抗力（天氣、班機延誤等）導致無法使用，非商家過失
- 客人自行操作不當或忘記取消訂單導致問題
- 客人單純詢問流程/政策，商家依規則正常說明（非互動品質問題）
- 客人兌到手的票券/贈品可正常使用，但覺得換算價值不如預期划算

**❌ 常見誤判**

- 頁面本身確實模糊沒寫清楚，客人才誤會→歸商品內容，非客人理解
- 供應商實際執行縮水/沒做到頁面承諾→歸供應商履約，非客人理解
- 抱怨實體品質本身差→歸商品品質，非客人理解
- 客服互動本身有問題（回應慢/態度差/不合理拖延）→歸客服營運，非客人理解
- 核銷/兌換操作本身卡關（非客人理解錯誤）→歸平台與系統，非客人理解
- 天候引發的現場處置導致悶熱/設備不可用(封艙/收帆/加蓋)→設施體驗差歸商品品質，本域僅收外力本身的影響
- 商家對外力事件的處置有明確失職（未告知/無備案/強行出團）→歸供應商履約 C-3-6，本域僅收外力本身

**⛔ 明確禁止**

- 客服互動品質問題（回應慢/拖延/態度差）禁止歸客人理解
- 頁面本身模糊不清時禁止歸客人理解（應先判商品內容）
- 有具體瑕疵事實（商品損壞/供應商未履約）時，禁止歸客人理解（應先判 C-2/C-3）
- 供應商明知風險仍未告知/未應變（有失職事實）→ 歸 C-3-6，禁全歸外力

**L2 面向**

| code | 面向 | 定義 |
|---|---|---|
| C-6-1 | 個人因素 | 因旅客自身狀況或操作導致用不到/體驗打折，商品與供應商正常：遲到睡過頭、身體不適、自行改變計畫，或自己按錯步驟/裝置設定問題（自承是自己弄錯）；核心是「旅客自身狀況與操作」。非外力事件→C-6-4/C-6-5、非供應商爽約→C-3-2、非商品/憑證真故障→C-2/C-4、非沒讀說明或下單選錯→C-6-6 |
| C-6-2 | 價值感落差 | 行程/商品依約正常完成、頁面費用與內容已寫清楚，旅客僅覺得整體「不值這個價」：嫌貴、CP值低、價值與價格不相稱、兌到手的票券/贈品換算價值不如預期划算；核心是「拿價格對比價值的主觀評價」（評論明確提及價格/票價/費用/划算與否）。頁面費用資訊沒寫清→C-1-3、現場臨時加收費→C-3-4、被話術施壓消費→C-3-7、與價格無關的內容落差→C-6-3。 |
| C-6-3 | 內容期待落差 | 行程/服務依約正常完成、無取消縮水或與頁面不符，旅客僅憑過往經驗或想像比較，覺得規模比想像小、內容沒想像豐富、氛圍景色普通、不如別的地方好玩，純屬主觀心理落差（評論聚焦價格對比→歸 C-6-2）。含依表執行的行程節奏不滿：嫌停留時間短、行程趕、拉車遠、購物站多但頁面已載明（評論無執行偏離表定的事實）——此類一律本類，非 C-3-3。落差主因是天氣/天災/公休等外力（起霧下雨看不到景、設施因天候關閉）→ 歸 C-6-4/C-6-5，非本類。 |
| C-6-4 | 天候與自然因素 | 天候/天災/自然條件（下雨/起霧/颱風/地震/退潮/季節/能見度/花況動物不賞臉）導致行程取消、設施停開或體驗打折，且商家已合理應變或無應變義務；核心是「天然外力不可控、非商家過失」。鏡像判準：外力事件本身歸本類；商家明知天候風險仍未告知/未取消/應變失當（有失職事實）→ 歸 C-3-6，可並存。頁面未揭露天候風險/保險→C-1-6、非天候的外部事件（疫情/罷工/場館關閉）→C-6-5。 |
| C-6-5 | 外部突發事件 | 非天候的外部突發事件（疫情/防疫政策、第三方運輸罷工延誤、場館臨時關閉、公共事件/管制）導致行程取消、縮減或改期，且商家已依規則協助或無應變義務；核心是「外部事件不可控、非商家過失」。鏡像判準：事件本身歸本類；商家明知事件仍未告知/應變失當 → 歸 C-3-6，可並存。天候/天災類→C-6-4。 |
| C-6-6 | 資訊誤讀 | 頁面/憑證資訊本身正確清楚，旅客自己沒看仔細或會錯意：下單選錯天數/張數/版本、誤解涵蓋範圍，或已載明的兌換步驟/需自備配件/使用須知沒讀到；核心是「資訊在但旅客沒吸收」（含自承沒看與下單操作疏失）。頁面本身語意不清/沒寫→C-1；旅客未自承沒看、僅現場被規則卡住→C-4-2（不預設旅客已理解）；系統重複扣款→C-5。 |
