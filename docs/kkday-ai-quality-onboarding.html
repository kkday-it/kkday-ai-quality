上手指南：clone 到跑出與我相同的專案

一條指令啟動（含依賴自動安裝），數據從公司 Google Drive 下載資料包後、在前台「資料導入」上傳。

git clone start.sh 本地上傳資料包 localhost:5273

主流程（4 步）

本機只需 git + 容器引擎（推薦免費開源的 colima）；不用裝 python/node/pnpm/PostgreSQL，全在容器內。

1

Clone 專案

取得原始碼並進入專案根目錄。

git clone https://github.com/kkday-it/kkday-ai-quality.git cd kkday-ai-quality
2

一條指令啟動（純 Docker · 空庫也能跑）

start.sh 偵測 Docker（未啟動會自動啟動），再 docker compose 起全服務。首次 build 較久，之後秒起。

./scripts/dev/start.sh
  • 偵測 Docker → 未啟動則自動啟動並等待就緒
  • 容器內起 PostgreSQL + 後端 :8100 + 前端 :5273 + 所有依賴
  • 改碼即生效（uvicorn --reload + vite HMR）；Ctrl-C 停止全部

沒裝容器引擎？推薦 colima（免費開源）：brew install colima docker docker-compose；start.sh 會自動 colima start。也支援 OrbStack / Docker Desktop。

3

註冊並登入

打開前端，在登入頁註冊帳號（空庫還沒有帳號）。用下面這組 email 註冊會自動成為 admin（在 roles.json 白名單）。

open http://localhost:5273
  • 範例帳號：email alvin.bian@kkday.com、密碼 123456（→ 自動 admin）
  • 密碼是你註冊時自己設的；用此 email 才有 admin 權限（規則發布 / 資料導入）

後端 Swagger：http://localhost:8100/docs

4

上傳資料包載入全部數據

先從 Google Drive 下載資料包 zip，再到「配置 › 資料導入」分頁上傳，即可載入全部數據。

資料包下載：Drive › kkday-ai-quality › datapacks

  • 點「選擇資料包 zip…」選下載的 kkday-ai-quality-datapack-*.zip → 自動校驗預覽（每表覆蓋筆數、schema 綠燈）
  • 輸入 REPLACE-ALL-DATA → 按「確認匯入」→ 看逐表進度
  • 不勾「納入帳號 / 機密表」→ 保留你註冊的 alvin.bian@kkday.com 帳號，只灌業務數據

匯入為覆蓋式：清空並以資料包重建各表。同一分頁也能「導出資料包」把當前數據下載成 zip。

資料包放哪（公司 Drive 統一交換）

資料包放在公司 Google Drive 的 kkday-ai-quality › datapacks 資料夾統一交換。

產生 / 更新資料包

前台「配置 › 資料導入 › 導出資料包」（逐表進度）下載，或 CLI python scripts/tools/dump_datapack.py；再上傳到 Drive › datapacks。

載入資料包

從 Drive › datapacks 下載 → 「配置 › 資料導入」上傳 → REPLACE-ALL-DATA → 匯入。

生產部署（選用）

上面主流程是 dev（hot reload）；正式部署走生產 compose（多 worker + nginx 靜態）：

AIQ_JWT_SECRET=<your-secret> docker compose up -d --build

前端 → http://localhost:8080；生產須提供 AIQ_JWT_SECRET，無 hot reload。

速查

後端 :8100 · /docs 前端(dev) :5273 前端(生產) :8080 DB kkdb_ai_quality

卡住時

  • start.sh 說找不到容器引擎 → 裝 colima（brew install colima docker docker-compose）再重跑。
  • 首次很久 → 在 build image（裝依賴），正常；之後秒起。
  • 畫面沒數據 → 到「配置 › 資料導入」上傳資料包 zip。
  • 改依賴（pyproject/package.json）沒生效 → docker compose -f docker-compose.dev.yml up --build 重建。
  • 登不進 → 空庫時先在登入頁註冊帳號。
