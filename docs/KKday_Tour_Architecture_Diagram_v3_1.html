系統定位說明
本系統對「內容治理問題」（第 1 / 2 / 3A 層）是主責者：偵測、判定、計點，輸出至 SCM2.0（供應商看）與 Be2（AM/BD 看），UI/UX 呈現由各後台負責。
本系統對「非內容服務履約問題」（第 3B 層）是感知者與通報者：偵測到問題並觸發通報，後續處理需客服 SOP 與供應鏈管理機制協作。
判定規則的研發與驗證在本系統內由 Tour PM + RD 執行，確認精準度後提交給審品系統作為正式規則執行，本系統是規則的制定者與實驗室，審品系統是正式執行環境。

系統定位一句話
本系統是 Tour 內容治理的感知延伸與執行閉環——規則的制定者、用戶感知的捕捉者、問題的判定者。
它只取用它需要的資料，只輸出它該輸出的結果，不介入任何其他系統的營運流程。
