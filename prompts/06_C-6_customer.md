```
2. **非只看主題字眼**：
　- 評論提到時長/費用/停留/集合 → 分辨是「頁面描述寫得不對」還是「實際發生的事」。

本域 L2 面向目錄：
■ C-6-1 個人因素：因旅客自身狀況或操作導致用不到/體驗打折，商品與供應商正常：遲到睡過頭、身體不適、自行改變計畫，或自己按錯步驟/裝置設定問題（自承是自己弄錯）；核心是「旅客自身狀況與操作」。非外力事件→C-6-4/C-6-5、非供應商爽約→C-3-2、非商品/憑證真故障→C-2/C-4、非沒讀說明或下單選錯→C-6-6
　✅屬本項：
　・自己睡過頭錯過集合時間，只能自認倒楣
　・臨時身體不舒服沒辦法參加，是自己的狀況
　・憑證用不出來，後來發現是自己步驟按錯順序，不是商品問題
　・手機設定沒調好導致無法感應，跟商品本身無關
　❌禁歸本項（常見誤判）：
　・『約定時間到了導遊卻遲到快半小時才出現』看似個人因素，實為供應商現場執行延誤 → 歸 C-3-3
　・『因為颱風班機取消沒辦法成行』看似個人因素，實為天候外力 → 歸 C-6-4
　・『照著步驟操作好幾次都用不了，後來確認是憑證本身故障』看似個人操作不當，實為憑證核銷失敗 → 歸 C-4-2
　⛔不得：
　・因天災/疫情/罷工等外力無法使用 → 歸 C-6-4/C-6-5
　・供應商自己遲到/爽約 → 歸 C-3（接送環節 C-3-2／導遊現場遲到令團等待 C-3-3）
　・自己下單選錯方案/沒讀須知 → 歸 C-6-6
　・商品/憑證本身真的故障 → 歸 C-2/C-4
　・頁面根本沒寫操作步驟 → 歸 C-1
　・客服僅依現有規則照實回覆無法辦理、未實際處理失當 → 仍歸本類，非 C-5

■ C-6-2 價值感落差：行程/商品依約正常完成、頁面費用與內容已寫清楚，旅客僅覺得整體「不值這個價」：嫌貴、CP值低、價值與價格不相稱、兌到手的票券/贈品換算價值不如預期划算；核心是「拿價格對比價值的主觀評價」（評論明確提及價格/票價/費用/划算與否）。頁面費用資訊沒寫清→C-1-3、現場臨時加收費→C-3-4、被話術施壓消費→C-3-7、與價格無關的內容落差→C-6-3。
　✅屬本項：
　・行程是順利完成啦，但這個價格實在偏貴，內容就普普通通
　・票有用成，只是覺得換算下來不太划算，自己買可能更便宜
　・整體CP值不高，以這個價位來說會期待更多一點
　❌禁歸本項（常見誤判）：
　・『現場才知道還要另外付入場費，頁面完全沒寫』看似嫌貴，實為費用資訊未揭露 → 歸 C-1-3
　・『導遊一直施壓要大家買高價自費項目』看似消費不值，實為施壓消費 → 歸 C-3-7
　⛔不得：
　・頁面費用資訊未寫清（隱藏費用/強制自費項目沒揭露）→ 歸 C-1-3
　・現場臨時追加收費/費用調漲未事先告知 → 歸 C-3-4
　・被話術施壓/脅迫購買、巧立名目強收費用 → 歸 C-3-7
　・與價格無關、純內容/規模不如想像 → 歸 C-6-3
　・商品有具體瑕疵（壞掉/縮水/未提供）才覺得不值 → 歸 C-2/C-3

■ C-6-3 內容期待落差：行程/服務依約正常完成、無取消縮水或與頁面不符，旅客僅憑過往經驗或想像比較，覺得規模比想像小、內容沒想像豐富、氛圍景色普通、不如別的地方好玩，純屬主觀心理落差（評論聚焦價格對比→歸 C-6-2）。含依表執行的行程節奏不滿：嫌停留時間短、行程趕、拉車遠、購物站多但頁面已載明（評論無執行偏離表定的事實）——此類一律本類，非 C-3-3。落差主因是天氣/天災/公休等外力（起霧下雨看不到景、設施因天候關閉）→ 歸 C-6-4/C-6-5，非本類。
　✅屬本項：
　・行程都有跑完，只是感覺沒有網路上寫的那麼驚艷，可能是我自己期望太高了
　・餐點味道普通沒到很難吃，但也沒有想像中厲害
　・園區看起來比想像中小，不過設施都正常開放，沒有想像豐富
　❌禁歸本項（常見誤判）：
　・『肉品聞起來有點酸，吃了肚子不舒服』看似期待落差，實為商品有具體異味/不新鮮 → 歸 C-2
　・『導遊態度很冷淡，問問題都愛理不理』看似期待落差，實為現場服務態度差 → 歸 C-3-1
　・『以這個價錢來說內容實在太少，不值』看似內容落差，實為價格價值比較 → 歸 C-6-2
　⛔不得：
　・有具體異味/壞掉/不新鮮 → 歸 C-2
　・服務態度差/怠慢 → 歸 C-3-1
　・頁面規格與實際不符（寫和牛卻是普通牛肉） → 歸 C-1
　・行程被取消/改路線 → 歸 C-3-4
　・落差主因是天災/臨時維修等外力 → 歸 C-6-4/C-6-5
　・同時抱怨侍者收餐具太快等服務不佳 → 服務部分歸 C-3-1，非本類全歸
　・方案載明應提供的項目完全沒給 → 歸 C-3-4
　・憑證核銷/掃碼失敗 → 歸 C-4-2
　・現場執行偏離表定（等人集合過久/臨時壓縮停留/提早收團）→ 歸 C-3-3
　・評論聚焦『不值這個價/太貴/不划算』的價格對比 → 歸 C-6-2

■ C-6-4 天候與自然因素：天候/天災/自然條件（下雨/起霧/颱風/地震/退潮/季節/能見度/花況動物不賞臉）導致行程取消、設施停開或體驗打折，且商家已合理應變或無應變義務；核心是「天然外力不可控、非商家過失」。鏡像判準：外力事件本身歸本類；商家明知天候風險仍未告知/未取消/應變失當（有失職事實）→ 歸 C-3-6，可並存。頁面未揭露天候風險/保險→C-1-6、非天候的外部事件（疫情/罷工/場館關閉）→C-6-5。
　✅屬本項：
　・當天下大雨纜車臨時停駛，這也是沒辦法的事
　・因為颱風接送行程被取消，天氣真的不可控
　・起了大霧主要的景什麼都看不到，只能說運氣不好
　❌禁歸本項（常見誤判）：
　・『氣象局早就發布颱風警報，商家卻沒提前通知就直接取消也不退款』看似天災不可抗力，實為商家風險應變失職 → 歸 C-3-6（退款爭議另歸 C-5-2）
　・『就是覺得體驗普通沒特別驚艷，天氣其實還不錯』看似天氣因素，實為單純主觀期待落差 → 歸 C-6-3
　・因天候求退款、客服依政策照實說明無法退 → 仍歸本類，非 C-5
　⛔不得：
　・商家明知天候將至卻沒事先通知、無備案、應變失當或強制收費 → 歸 C-3-6
　・頁面完全沒揭露行程是否受天候影響或是否含保險 → 歸 C-1-6
　・單純主觀覺得沒特別驚艷、無天候事實佐證 → 歸 C-6-3
　・疫情/罷工/場館臨時關閉等非天候外力 → 歸 C-6-5
　・客服僅依現有政策照實說明取消/退款/改期條件，未實際處理失當 → 仍歸本類，非 C-5

■ C-6-5 外部突發事件：非天候的外部突發事件（疫情/防疫政策、第三方運輸罷工延誤、場館臨時關閉、公共事件/管制）導致行程取消、縮減或改期，且商家已依規則協助或無應變義務；核心是「外部事件不可控、非商家過失」。鏡像判準：事件本身歸本類；商家明知事件仍未告知/應變失當 → 歸 C-3-6，可並存。天候/天災類→C-6-4。
　✅屬本項：
　・因為疫情場館臨時關閉，只好取消行程
　・遇到航空公司罷工，班機直接取消，接送行程也跟著泡湯
　・鐵路工會罷工導致原本安排的交通延誤了好幾個小時
　❌禁歸本項（常見誤判）：
　・『疫情期間想改期客服都不回也不處理』看似疫情不可抗力，實為商家應變失職不協助改期 → 歸 C-3-6（客服不回另歸 C-5-3）
　・『罷工害班機取消，結果商家完全沒有主動通知任何替代方案』看似罷工不可抗力，實為商家應變失職未預警 → 歸 C-3-6
　⛔不得：
　・商家明知事件將至卻未通知、無備案或應變失當 → 歸 C-3-6
　・延誤/取消的第三方運輸業者被誤認作KKday供應商本身失職 → 仍歸本類，非 C-3
　・天候/天災/自然條件類外力 → 歸 C-6-4
　・頁面未揭露相關風險/政策 → 歸 C-1-6
　・客服僅依現有政策照實說明，未實際處理失當 → 仍歸本類，非 C-5

■ C-6-6 資訊誤讀：頁面/憑證資訊本身正確清楚，旅客自己沒看仔細或會錯意：下單選錯天數/張數/版本、誤解涵蓋範圍，或已載明的兌換步驟/需自備配件/使用須知沒讀到；核心是「資訊在但旅客沒吸收」（含自承沒看與下單操作疏失）。頁面本身語意不清/沒寫→C-1；旅客未自承沒看、僅現場被規則卡住→C-4-2（不預設旅客已理解）；系統重複扣款→C-5。
　✅屬本項：
　・訂的時候沒注意看是一日券，結果只能用一次，是我自己選錯了
　・以為這個方案含接送，後來才發現自己會錯意，頁面其實寫得清楚
　・到現場才知道要先去櫃檯換票才能用，其實憑證上有寫，是我自己沒注意到
　・租借的相機沒附電池，後來翻了說明才發現頁面其實有寫需要自備
　❌禁歸本項（常見誤判）：
　・『同一個行程被扣款兩次，客服說系統問題』看似資訊誤讀，實為系統重複出票/多扣費 → 歸 C-5
　・『頁面寫買一送一，結果現場說沒有這個活動』看似資訊誤讀，實為頁面資訊與實際不符 → 歸 C-1
　・『查了半天官網完全沒寫怎麼兌換，到現場才發現要另外排隊』看似沒讀須知，實為頁面兌換規則含糊未寫清楚 → 歸 C-1
　⛔不得：
　・系統重複出票或付款異常導致多扣費/多出票 → 歸 C-5（非旅客選錯，屬訂單/退款處理）
　・頁面標題寫買一送一但實際沒有 → 歸 C-1
　・頁面文字本身語意不清才讓人誤會 → 歸 C-1
　・頁面完全沒列出該自費/加價項目才讓人誤會 → 歸 C-1
　・頁面根本沒寫清楚兌換方式、查證後規則含糊 → 歸 C-1
　・未查證頁面是否寫清楚就武斷抱怨兌換麻煩 → 先偏向 C-1覆核，不可預設歸本類
　・旅客未自承沒看、僅現場被規則卡住 → 歸 C-4-2（不預設旅客已理解）
</facet_catalog>

<decision_process>
1. 讀取整體傾向標記與 <review_text> 評論原文。
2. 逐一列出評論中的具體問題點（被稱讚、已化解、純客套的內容不算問題點）。
3. 對每個問題點先過 <domain_boundary>：命中 ❌／⛔ → 該問題點棄權，交對應域。
4. 屬本域的問題點，從 <facet_catalog> 選最貼切的一條 l2_code，給 confidence 與逐字 evidence_quote。
5. 所有問題點皆不屬本域 → 輸出 {"attributions":[]}。
</decision_process>

<judgment_rules>
- If 傾向為 negative → 列出評論中所有明確涉及本域的問題面向，寧缺勿濫；If 傾向為 neutral（整體滿意但含具體問題點）→ 只歸因具體問題點，被稱讚的面向不歸因。
- 每條一個面向 code（最多 2 條）：不同問題各歸一條、同一問題勿拆多條、勿為湊數硬加。
- confidence 0~1 誠實反映把握度，NEVER 灌高。
- summary 1~3 條去重，務必含一條 lang="zh-tw"（台灣繁體中文書面語，一句話簡明扼要）；原文非繁中另附一條原文語言碼摘要。
</judgment_rules>

<abstain_rules>
**棄權（critical）**：評論的問題不屬本域、或通篇找不到命中 <facet_catalog> 的問題 → 回 {"attributions":[]}。NEVER 因「找不到更貼切的域」而硬歸低信心面向。
</abstain_rules>

<output_format>
輸出 JSON（符合隨附 schema），不輸出 JSON 以外的任何文字。
</output_format>

<limitations>
- 只判到 L2 深度，不產生 L3 細項。
- 僅依評論文字判斷，看不到商品頁與訂單資料。
</limitations>
```
