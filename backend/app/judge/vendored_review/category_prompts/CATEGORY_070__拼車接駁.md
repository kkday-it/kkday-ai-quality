## 判準（分類與文案一致性）
判斷**本欄位文字在賣的「商品主體 / 主要活動」**，是否落在本商品分類**「拼車接駁」**的涵蓋範圍。

**做法**：
1. 先從本欄位文字抓出「這個商品主要在賣什麼、提供什麼體驗」（主體）。
2. 再對照下方「本商品分類定義」的涵蓋範圍。
3. 只有主體**明顯落在範圍外、且不屬合理加值**時，才算分類錯置。

### 本商品分類定義（判斷基準）
分類路徑：交通 > 包車 > 拼車接駁（以最後一層「拼車接駁」為主）
- T1「交通」定義：此類別涵蓋各種『移動／運輸』相關商品，如機場接送、租車與包車、鐵路與巴士票券、交通卡與通票、渡輪與船票等。重點在把旅客從一地運送到另一地、或提供交通工具／票證，而非目的地的景點或活動。
- T2「包車」定義：提供私人包車和專車服務的預訂，旨在提供個人或小團體專屬的交通選擇。 通常包含專屬車輛以及私人司機。 這些服務提供了更為靈活和舒適的交通解決方案，適用於個人或小團體需求。
- T3「拼車接駁」（主）定義：拼車接駁是指多名乘客共享同一輛車，按照預定的路線或站點進行接送。這種服務通常以較經濟實惠的價格提供，因為多名乘客共享成本。拼車接駁常用於特定區域或路線，以滿足多人的交通需求。

### 命中 / 不命中
**命中（matched=true）**：本欄位描述的商品主體與本分類**明顯矛盾**（例：分類為「拼車接駁」，但文字主體明顯在賣與此分類無關的東西）。

**不命中（Pass）**：
- 主體與「拼車接駁」相符，或屬其合理涵蓋範圍（含常見加值，如交通接送、餐食、導覽、購物等）。
- 本欄位**主要是退費／集合時間／注意事項等非描述商品主體**的內容，或資訊太少（如僅商品名 3 字）
  → 不足以判定分類，**從寬 matched=false**，避免誤判。

## 輸出格式（強制）
只輸出合法 JSON，不可有 markdown／前言／註解。依本次列出的規則逐條判斷，彙整成「單一結果」：

```json
{
  "matched": <boolean>,
  "hits": [
    {
      "rule_oid": <違反的規則 oid，須為本次列出的規則之一>,
      "field": "<content_json 裡的欄位 key，如 product.prod_name.zh-tw>",
      "actual": "<命中文字片段，含前後文 10~30 字，必須是欄位原文，不可杜撰>",
      "reason": "<違規類型 + 判斷依據，一句話>"
    }
  ]
}
```

原則：
- 只輸出上述 JSON 物件本身；`matched` 為頂層布林，`rule_oid` 放在每個 hit 內。
- `actual` 引用欄位原文語系，`reason` 用同一語系回答。
- recall 優先：拿不準傾向 `matched=true` 並指出疑慮，不要漏放。
- 只依本次列出的規則判定；未違規時 `matched=false` 且 `hits=[]`。
