## 判準（分類與文案一致性）
判斷**本欄位文字在賣的「商品主體 / 主要活動」**，是否落在本商品分類**「工作坊」**的涵蓋範圍。

**做法**：
1. 先從本欄位文字抓出「這個商品主要在賣什麼、提供什麼體驗」（主體）。
2. 再對照下方「本商品分類定義」的涵蓋範圍。
3. 只有主體**明顯落在範圍外、且不屬合理加值**時，才算分類錯置。

### 本商品分類定義（判斷基準）
分類路徑：行程＆體驗 > 文化體驗 > 工作坊（以最後一層「工作坊」為主）
- T1「行程＆體驗」定義：此類別涵蓋由供應商規劃、帶有行程安排或活動體驗的商品，如觀光行程、戶外與運動活動、文化體驗、課程與工作坊等。重點在『安排好的活動／體驗過程』，通常含導覽、教學或帶領，而非單純的入場票券或單一設施使用。
- T2「文化體驗」定義：涵蓋了各種文化體驗和傳統習俗學習機會。 讓旅客深入體驗當地文化和歷史。 如文化體驗、工作坊和課程，包括藝術工作坊、烹飪課程和民俗文化示展示或體驗。
- T3「工作坊」（主）定義：各種手作體驗和線上課程，旨在提供文化和技能的學習機會。 包括手作工作坊、藝術課程、手工藝品製作和其他技能體驗。 這些活動讓參與者可以主動參與，體驗在地技藝，並深入了解當地文化。
- 規格參考（Tier4，僅補充分類涵蓋範圍，非必須符合項）：類型：手作體驗、線上課程

### 命中 / 不命中
**命中（matched=true）**：本欄位描述的商品主體與本分類**明顯矛盾**（例：分類為「工作坊」，但文字主體明顯在賣與此分類無關的東西）。

**不命中（Pass）**：
- 主體與「工作坊」相符，或屬其合理涵蓋範圍（含常見加值，如交通接送、餐食、導覽、購物等）。

## 輸出格式（強制）
只輸出合法 JSON，不可有 markdown／前言／註解。依本次列出的規則逐條判斷，彙整成「單一結果」：

```json
{
  "matched": <boolean>,
  "hits": [
    {
      "rule_oid": <違反的規則 oid，須為本次列出的規則之一>,
      "field": "<content_json 裡的欄位 key，如 product.prod_name.zh-tw>",
      "actual": "<命中文字片段，含前後文 10~30 字，必須是欄位原文，不可杜撰>",
      "reason": "<違規類型 + 判斷依據，一句話>"
    }
  ]
}
```

原則：
- 只輸出上述 JSON 物件本身；`matched` 為頂層布林，`rule_oid` 放在每個 hit 內。
- `actual` 引用欄位原文語系，`reason` 用同一語系回答。
- recall 優先：拿不準傾向 `matched=true` 並指出疑慮，不要漏放。
- 只依本次列出的規則判定；未違規時 `matched=false` 且 `hits=[]`。
