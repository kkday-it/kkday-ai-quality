# 請檢查新商品主要語言與撰寫語言是否相符

## 規則
- **編號**：R25050622（oid=72）　**風險等級**：R4
- **規則內容**：新商品主要語言與撰寫語言是否相符
- **判準（通知文案，僅供參考）**：主要語言與實際撰寫語言不符。

## 程式碼已過濾（不需 LLM 再判）
- `是否新品 為是`。前提已成立。

## LLM 判準（語言辨識，本規則適合走 LLM）
判斷**本欄位文字實際使用的語言**，是否與商品**宣告的主要語言**一致。
- 系統會把「宣告主要語言」放進中繼資料（即本次審核語系 `語系=`）。支援語系：
  zh-tw、zh-hk、zh-cn、en、ja、ko、th、vi。

**命中（matched=true）**：本欄位主體文字的語言與宣告主要語言**明顯不符**，例如：
- 宣告主要語言為日文（ja），但 prod_desc 整段是繁體中文；
- 宣告 zh-tw，但內容整段為英文。

**不命中（Pass）**：
- 語言相符。
- **少量混入**屬正常，不算違規：專有名詞、地名、品牌、外語術語、單句英文標語、價格單位等。
- 欄位為空或僅符號/數字 → matched=false（無從判斷）。

**判斷重點**：看**主體敘述語言**，不要因夾雜少數外語詞就判不符。reason 說明「本欄位主體為 X 語，與宣告主要語言 Y 不符」。

## 輸出格式（強制）
只輸出合法 JSON，不可有 markdown／前言／註解。依本次列出的規則逐條判斷，彙整成「單一結果」：

```json
{
  "matched": <boolean>,
  "hits": [
    {
      "rule_oid": <違反的規則 oid，須為本次列出的規則之一>,
      "field": "<content_json 裡的欄位 key，如 product.prod_name.zh-tw>",
      "actual": "<命中文字片段，含前後文 10~30 字，必須是欄位原文，不可杜撰>",
      "reason": "<違規類型 + 判斷依據，一句話>"
    }
  ]
}
```

原則：
- 只輸出上述 JSON 物件本身；`matched` 為頂層布林，`rule_oid` 放在每個 hit 內。
- `actual` 引用欄位原文語系，`reason` 用同一語系回答。
- recall 優先：拿不準傾向 `matched=true` 並指出疑慮，不要漏放。
- 只依本次列出的規則判定；未違規時 `matched=false` 且 `hits=[]`。
