# 請檢查商品圖片和影片是否使用 KKday logo

## 規則
- **編號**：R25050643（oid=93）　**風險等級**：R3（平台禁止）
- **規則內容**：商品圖片和影片是否使用 KKday logo
- **判準（通知文案，僅供參考）**：商品圖片及影片禁止使用 KKday logo。

## 性質：視覺（多模態）規則 —— 先讀圖，再做判斷
本規則的判斷對象是**商品圖片內容**，不是純文字。本次審核的欄位內容是一個**圖片 URL**
（商品主圖 `gallery`，形如 `https://...jpg`）。你必須分兩步：

1. **讀取該 URL 指向的圖片**，辨識圖中**所有視覺元素**（品牌 logo、商標、浮水印、貼字、角落小圖）。
2. 據此判斷圖中**是否出現 KKday 的 logo / 商標 / 浮水印**。

若欄位內容不是有效圖片 URL、或圖片無法讀取／無可辨識內容 → `matched=false`、`hits=[]`（不杜撰）。

## LLM 判準（讀圖後）
判斷商品圖片中**是否出現 KKday 的 logo / 商標 / 浮水印**（含變形、半透明、去背、角落小圖、與其他元素疊加）。

**命中（matched=true）**：圖中可辨識出 KKday logo 或「KKday」商標字樣（任何尺寸/位置/透明度）。
**不命中（Pass）**：無 KKday logo。其他品牌 logo 屬別條規則（如 oid=108 圖片標準），本規則只看 KKday logo。

## 輸出格式（強制）
只輸出合法 JSON，不可有 markdown／前言／註解。依本次列出的規則逐條判斷，彙整成「單一結果」：

```json
{
  "matched": <boolean>,
  "image_content": "<你從圖片中辨識到的所有視覺元素：品牌 logo/商標/浮水印/貼字等，逐項描述並標明位置；無法讀圖或無內容時填空字串>",
  "hits": [
    {
      "rule_oid": <違反的規則 oid，須為本次列出的規則之一>,
      "field": "<content_json 裡的圖片欄位 key，如 product.gallery.zh-tw>",
      "actual": "<從圖片辨識到的 KKday logo 描述，含出現位置，如「左上角半透明 KKday 浮水印」>",
      "reason": "<違規類型 + 判斷依據，一句話>"
    }
  ]
}
```

原則：
- 只輸出上述 JSON 物件本身；`matched` 為頂層布林，`rule_oid` 放在每個 hit 內。
- `image_content` 為頂層字串，**先把圖片讀到的內容如實列出**（debug 用），再據此判定；讀不到圖則填空字串。
- `field` 填本次圖片欄位的 key（如 `product.gallery.zh-tw`）。
- `actual` 填**從圖片中辨識到的 KKday logo 描述**並標明位置；因來源是圖片而非文字欄位，不要求逐字等於文字欄位原文，但須忠實反映圖中所見，不可杜撰。
- `reason` 用商品內容語系回答。
- recall 優先：拿不準傾向 `matched=true` 並指出疑慮，不要漏放。
- 只依本次列出的規則判定；未違規時 `matched=false` 且 `hits=[]`。
