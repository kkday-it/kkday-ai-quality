# 商品圖片中含導外訊息

## 規則
- **編號**：R2505065（oid=55）　**風險等級**：R3（平台禁止）
- **規則內容**：商品圖片中含導外訊息
- **判準（通知文案，僅供參考）**：商品圖禁止出現品牌名、電話、官網、「更多優惠請上粉絲團」等廣告字樣，請置換圖片。

## 性質：視覺（多模態）規則 —— 先讀圖，再做導外判斷
本規則的判斷對象是**商品圖片內容**，不是純文字。本次審核的欄位內容是一個**圖片 URL**
（商品主圖 `gallery`，形如 `https://...jpg`）。你必須分兩步：

1. **讀取該 URL 指向的圖片**，辨識圖中**所有文字**（浮水印、貼字、海報字、橫幅、
   QR code 旁的說明文字）與**視覺元素**（品牌 logo、社群圖示、QR code、聯絡資訊框）。
2. 把第 1 步從圖片讀到的內容，**比照「商品文案含導外訊息」（oid 56）的同一套判準**做導外偵測。

若欄位內容不是有效圖片 URL、或圖片無法讀取／無可辨識內容 → `matched=false`、`hits=[]`（不杜撰）。

## LLM 判準（讀圖後，與 oid 56 同一套導外判準）
判斷圖片中是否出現**引導旅客脫離 KKday 聯繫或下單**的導外訊息：

> 參考觸發詞（**非封閉清單**，共 25 項）：`[line.me、id:@、ID:@、LINE群組、Line群組、line群組、群組、官方帳號、官方line、官方Line、官方LINE、line帳號、Line帳號、LINE帳號、加line、加Line、加LINE、Line：@、LINE ID、Line ID、line ID、facebook、klook、KLOOK、客路]`。導外手法千百種，仍須語意涵蓋未列出的變體（諧音、全形、拆字、外語、以 logo 或 QR code 形式呈現）。

**命中（matched=true）**：圖中可辨識出下列任一（不論以文字浮水印、貼字、海報字或 logo / QR code 形式呈現）：
- 社群/通訊導流：LINE 帳號或群組（加 line、官方帳號、Line ID、`@xxx`、`line.me/...`）、FB / IG / WeChat 帳號或粉絲團、「更多優惠請上粉絲團」「掃碼加入」。
- 聯絡方式導外：電話、手機、Email、官網網址、指向站外的 QR code。
- 競品平台名稱／導購：Klook、客路 等 KKday 以外的訂購平台，或「請至 XX 平台預訂」。
- 品牌自有通路宣傳：引導至供應商自家官網／商城下單。
- 變體仍要抓：全形/半形混用、拆字（如「加 賴」）、諧音、外語寫法、品牌 logo 浮水印。

**不命中（Pass）**：
- 圖片僅為景點／體驗／商品實拍，無上述導外文字或競品標記。
- 圖中文字僅為體驗內容描述，或 KKday 自身客服／說明等內部資訊，無引導站外聯繫或下單意圖。
- 一般地名/品牌名僅作為體驗內容呈現，無引導聯繫或下單意圖。

**紀律**：拿不準是否構成導外時，偏向 `matched=true`；`reason` 說明命中的導外元素（哪個帳號／平台／聯絡方式）出現在圖片何處（如「右下角浮水印」「海報中央 QR code」）。

## 輸出格式（強制）
只輸出合法 JSON，不可有 markdown／前言／註解。依本次列出的規則逐條判斷，彙整成「單一結果」：

```json
{
  "matched": <boolean>,
  "image_content": "<你從圖片中辨識到的所有內容：可見文字（浮水印/貼字/海報字/QR 旁文字）＋視覺元素（logo/社群圖示/QR code/聯絡資訊框）。逐項描述並標明位置；無法讀圖或無內容時填空字串>",
  "hits": [
    {
      "rule_oid": <違反的規則 oid，須為本次列出的規則之一>,
      "field": "<content_json 裡的圖片欄位 key，如 product.gallery.zh-tw>",
      "actual": "<從圖片辨識到的導外文字或元素描述，含出現位置，如「右下角浮水印：官方LINE @abc123」>",
      "reason": "<違規類型 + 判斷依據，一句話>"
    }
  ]
}
```

原則：
- 只輸出上述 JSON 物件本身；`matched` 為頂層布林，`rule_oid` 放在每個 hit 內。
- `image_content` 為頂層字串，**先把圖片讀到的內容如實列出**（debug 用），再據此判導外；讀不到圖則填空字串。
- `field` 填本次圖片欄位的 key（如 `product.gallery.zh-tw`）。
- `actual` 填**從圖片中辨識到的導外文字或元素描述**並標明位置；因來源是圖片而非文字欄位，不要求逐字等於文字欄位原文，但須忠實反映圖中所見，不可杜撰。
- `reason` 用商品內容語系回答。
- recall 優先：拿不準傾向 `matched=true` 並指出疑慮，不要漏放。
- 只依本次列出的規則判定；未違規時 `matched=false` 且 `hits=[]`。
