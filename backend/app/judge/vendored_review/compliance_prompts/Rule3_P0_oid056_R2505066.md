# 商品文案中含導外訊息

## 規則
- **編號**：R2505066（oid=56）　**風險等級**：R3（平台禁止）
- **規則內容**：商品文案中含導外訊息
- **判準（通知文案，僅供參考）**：商品文案中含品牌名、電話、官網、「更多優惠請上粉絲團」等廣告宣傳字樣，請移除導外訊息。

## 性質：純文字（語意）規則
- 觸發條件 `商品全文 內文含下列任一詞` 為**字串閘門**，但詞表無法窮舉所有導外手法，仍須 LLM 語意涵蓋變體（諧音、全形、拆字、外語）。
- 判斷對象是**商品文案文字本身**（標題、描述、方案、購買須知、費用包含/不含等）。本規則在純文字流程即可執行。

## LLM 判準（本規則唯一要做的事）
判斷**本欄位文字**是否出現**引導旅客脫離 KKday 聯繫或下單**的導外訊息：

> 觸發詞（規則完整清單，共 25 項）：`[line.me、id:@、ID:@、LINE群組、Line群組、line群組、群組、官方帳號、官方line、官方Line、官方LINE、line帳號、Line帳號、LINE帳號、加line、加Line、加LINE、Line：@、LINE ID、Line ID、line ID、facebook、klook、KLOOK、客路]`。此為**非封閉清單**：導外手法千百種，仍須語意涵蓋未列出的變體（這正是要走 LLM、而非純字串比對的原因）。

**命中（matched=true）**：
- 社群/通訊導流：LINE 帳號或群組（加 line、官方帳號、Line ID、`@xxx`、`line.me/...`）、FB / IG / WeChat 帳號或粉絲團、「更多優惠請上粉絲團」「掃碼加入」。
- 聯絡方式導外：電話、手機、Email、官網網址、QR code 指向站外。
- 競品平台名稱／導購：Klook、客路、KKday 以外的訂購平台或「請至 XX 平台預訂」。
- 品牌自有通路宣傳：引導至供應商自家官網/商城下單。
- 變體仍要抓：全形/半形混用、拆字（如「加 賴」）、諧音、外語寫法。

**不命中（Pass）**：
- 純地名/品牌作為體驗內容描述，且無引導聯繫或下單意圖（如僅描述行經某景點）。
- 平台允許的內部資訊（KKday 自身客服/說明），非導向站外。
- 「群組」等詞出現在無導外語境（如「分組進行活動」），須看語境。

**紀律**：拿不準是否構成導外時，偏向 matched=true；`actual` 必須是欄位原文片段，`reason` 說明命中的導外元素（哪個帳號/平台/聯絡方式）。

## 輸出格式（強制）
只輸出合法 JSON，不可有 markdown／前言／註解。依本次列出的規則逐條判斷，彙整成「單一結果」：

```json
{
  "matched": <boolean>,
  "hits": [
    {
      "rule_oid": <違反的規則 oid，須為本次列出的規則之一>,
      "field": "<content_json 裡的欄位 key，如 product.prod_name.zh-tw>",
      "actual": "<命中文字片段，含前後文 10~30 字，必須是欄位原文，不可杜撰>",
      "reason": "<違規類型 + 判斷依據，一句話>"
    }
  ]
}
```

原則：
- 只輸出上述 JSON 物件本身；`matched` 為頂層布林，`rule_oid` 放在每個 hit 內。
- `actual` 引用欄位原文語系，`reason` 用同一語系回答。
- recall 優先：拿不準傾向 `matched=true` 並指出疑慮，不要漏放。
- 只依本次列出的規則判定；未違規時 `matched=false` 且 `hits=[]`。
