# app/api — FastAPI gateway

HTTP 邊界層：路由 → 委派 `app/core/db` + `app/judge`。薄層（業務邏輯下沉 core/judge），
啟動時 `db.init_db()`（create_all，冪等）+ `db.seed_rules_from_files()`。

| 項目 | 內容 |
|---|---|
| `main.py` | 僅 app 組裝：CORS + `db.init_db()`/`seed_rules_from_files()` + 掛載全部 router（端點實作分散於 `routers/`，各自帶完整 /api 路徑）。`docs_kwargs()` 環境閘：production 關閉 /docs /redoc /openapi.json（schema 面不對未認證流量公開），dev/SIT 保留。 |
| `routers/auth.py` | 當前身分＋權限清單唯讀端點（/api/auth：**me** / **permissions**）。去帳戶系統（2026-07-22）後本地模式無 register/login/登出/切換帳號，`auth.get_current_user` 回固定身分。GET `/api/auth/permissions` 回當前 user 的 business-key 權限清單（be2 `auth.business-list` **wire 契約 `{value, ttl}`**；前端快取層自記 `startTime` 成 be2 Cache wrapper 形狀，見 `permission.api.ts`），供前端 usePermission() / 選單過濾。 |
| `routers/inbound.py` | 資料錄入（/api/inbound：validate / upload / upload/stream SSE）+ 批次清單（/api/batches）。 |
| `routers/settings.py` | 設定（/api/settings：get/update/raw/test-llm）+ QC DB 連線測試（/api/datasource/qc-db/test）；含 `load_user_context` 守衛 + `_activate_settings`（contextvar 注入 judge 路徑）。 |
| `routers/findings.py` | 歸因人工動作（PATCH /api/findings/{id}/verdict 單筆判決〔confirmed/dismissed/new＝撤銷·同值冪等〕｜PATCH /api/findings/batch/verdict 批量判決〔勾選評論的全部歸因，單交易 diff〕｜備註 notes｜級聯樹 GET /api/findings/taxonomy-cascade〔L1→L2 巢狀，供歸因列表篩選 cascader〕，**需登入**·記操作者/時間 audit）+ 評論級歸因歷史（GET /api/attribution-history 時間軸〔prejudge/verdict/note 三類事件〕、POST /api/attribution-history/notes 評論級備註、GET /api/attribution-history/models 歷來初判過的模型清單〔篩選/導出下拉〕）。 |
| `routers/evidence.py` | 訂單佐證唯讀查詢（GET /api/evidence/{order_oid}）：與判決管線共用 `db/qc_evidence.get_evidence()`（快取/熔斷/PII 防線同一套），憑證走當前登入 user 的解析；非成功回 status 降級不走 5xx。 |
| `routers/problems.py` | 統一問題列表 / 縱覽聚合（/api/problems*，皆支援 model CSV 多選＝初判模型篩選·當前初判維度）+ 列表導出（POST /api/problems/export → 背景 job；欄尾附 C-1~C-6 六域命中欄〔符合/不符合·未初判留空〕＋「分類統計」「Prompts」工作表；`snapshot_model`＝輸出結果版本〔內容替換為該模型 attribution_history 最新快照〕；`compare_models`＝並排對比模型多選〔基準右側每模型附一組情緒/L1/L2 欄，值取該模型最新快照〕——兩者語義獨立可並用）。 |
| `routers/v1/` | 三 router 皆掛 `prefix="/prejudge"`、`__init__` 聚合於 `/api/v1`（2026-07-23 從單一 `prejudge.py` 拆成一 router 一領域，路徑不變）：`prejudge.py`（正式批量初判：啟動/筆數預覽 POST `/count`（與啟動同一套標的解析 `_resolve_target_ids`，可 `within_ids` 交集勾選範圍）/SSE `/stream`/暫停 `/pause`/恢復 `/resume`/停止 `/cancel` + 歸因歷史 GET `/runs`·`/runs/{job_id}`——run 級 LLM 使用紀錄，執行中列 overlay in-mem 即時進度；`LlmOverridesIn`/`PrejudgeIn`/`_resolve_target_ids` 為三領域共用契約，由另外兩檔直接 import 重用）｜`prompt_debug.py`（售後根因 ad-hoc 調試：GET `/prompt-debug/defaults` 回預設 Prompt/schema、POST `/prompt-debug/stream` SSE 結構化裁決，不落 attributions）｜`prompt_sandbox.py`（歸因列表「Prompt 測試」沙盒——`PromptSandboxIn` 繼承 `PrejudgeIn`：POST `/prompt-sandbox`（勾選 prompt 子集 ungated 跑，回 job_id）/ POST `/prompt-sandbox/count`（筆數預覽）/ GET `/prompt-sandbox/status`（輪詢進度）/ GET `/prompt-sandbox/runs`·`/prompt-sandbox/runs/{run_id}`（測試歷史列表/詳情，含完整 LLM log 快照回看，與正式初判歷史完全分離；stub 模式無條件拒跑，dev 亦不例外）。草稿閉環：啟動 body 另收 `drafts`（{rule_code: 草稿 md 全文}，送測前逐條 `prompt_source.validate` 強驗 fail-fast）與 `compare`（雙跑對比：每筆 baseline/draft 各跑一遍）；雙跑 run 詳情動態附等價性 `metrics`；GET `/prompt-sandbox/runs/compare?a=&b=`＝兩筆 run 按 source_id 對齊對比＋metrics（雙跑 run 取其 draft 變體，須定義於 `/runs/{run_id}` 之前避免路徑攔截））。 |
| `routers/rules.py` | 初判規則版本化 CRUD（/api/judge-rules：list/active/history/save/restore/reset + jsonschema 驗證）；POST `/export` 啟動初判 Prompt 包 zip 導出背景 job；Prompt 草稿（GET `/drafts` 草稿存在清單、GET/PUT/DELETE `/{code}/draft` 單支草稿——未入庫編輯中內容，供沙盒送測/雙跑對比，見 `core/db/prompt_drafts.py`；存草稿寬鬆只驗 text 非空，送測/入庫才強驗）；POST `/{code}/validate` dry-run 驗證 prompt md（不落庫，內容不合法回 200 `{valid:false,error}`）。**寫入端點（save/restore/reset×2）掛 `require_permission(judge-rule.version.manage)`（admin 級·403）**。 |
| `routers/exports.py` | 通用導出 job 端點（/api/exports：SSE `stream` 進度 / `download` 取檔 / `cancel` 停止），搭 `app/core/export_jobs` 全域 registry，問題列表 / Prompt 包 / 資料包導出共用。 |
| `routers/overview.py` | 質檢概覽真實指標（GET /api/overview/ai-judge：attributions 內容類占比月趨勢 + 總量；「縮窄真接」——外部系統指標不在此，前端維持示意）。 |
| `routers/admin_import.py` | 全庫資料包導出/匯入（/api/admin：POST `/export/start` 啟動導出背景 job〔逐表進度，復用通用 export_jobs，下載走 /api/exports/download〕；POST `/import/validate` 乾跑校驗、POST `/import` 確認匯入背景 job、GET `/import/stream` SSE），委派 `app/core/db/datapack` + `app/core/import_jobs` + `app/core/export_jobs`。匯入只灌白名單表·不執行 SQL；環境閘 `AIQ_ALLOW_DATA_IMPORT`（dev 開）。授權：匯入 `data.datapack.import`、導出 `data.datapack.export`——兩者現在 `default`（人人可用），經可替換權限框架（`app/core/permissions`）判定，日後要收緊只改 `config/global/permissions.json` 的 `default`。 |

> 認證：本地模式無 token（`get_current_user` 回固定身分）；be2 模式 Bearer header 驗簽（見 `core/auth_verifiers.py`）。capability-token 端點（prejudge / 導出 SSE `stream` / import `stream`）以 job_id 免 header，其餘（cancel / download / export / import）仍需認證。
> 授權：破壞性端點掛 `require_permission(key)`（`app/core/permissions`）——無角色，直接授予：rules write＝僅 `grants[email]`（`judge-rule.version.manage`）；findings 判決、inbound 上傳、problems 導出、datapack 導出/匯入、批量初判啟停（`prejudge.run`，消耗 LLM 額度）現行皆在 `default`（登入即可用）；problems / overview 讀端點掛登入最低門檻（`get_current_user`）。`default ∪ grants[email]` 映射見 `config/global/permissions.json`；provider 切換見 `auth.config.json`。
