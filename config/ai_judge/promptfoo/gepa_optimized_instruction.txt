任務說明：

請依據提供的『問題分類 L3 目錄』，將一則負向旅遊商品評論嚴格且精準地歸類到最貼切的一條 L3 code。  
本任務核心在於「嚴格且精準地判斷負向評論的主體問題域，並依據細項定義選出最貼切的 L3 code」，避免跨域誤判與模糊歸類，寧缺勿濫。

---

【輸入格式】  
- review：一則負向旅遊商品評論（中文）  
- catalog：包含多條 L3 code 及其定義的目錄，格式為「code | 分類層級›細項名稱 | 定義說明」

---

【任務目標】  
1. 判斷評論的「主體問題域」：  
   - 先判斷評論主要針對的領域（例如：商品內容、商品品質、供應商履約、使用體驗、客服營運、客人理解等）。  
   - 例如：  
     - 若評論是供應商未履約（如未成團、臨時取消、擅自更改行程或餐點等執行面問題），歸入「供應商履約」相關分類。  
     - 若評論是商品頁描述與實際不符，屬「商品內容」。  
     - 若是執行過程中未落實約定，屬「供應商履約」。  
     - 若評論涉及客服回應內容空洞、答非所問、AI客服鬼打牆，屬「客服營運›客服回應›AI鬼打牆」。  
     - 若評論是客服態度冷淡、推諉，屬「客服營運›客服態度›態度推諉」。  
     - 若評論是旅客自身期待與實際體驗落差，屬「客人理解›期待落差」。  

2. 嚴格依目錄細項邊界歸類：  
   - 僅能從目錄中選擇 code，且必須符合該 code 的定義與範圍。  
   - 若評論內容無法明確歸類到任何一條，請回空字串 ""（寧缺勿濫）。  
   - 不可模糊或跨域歸類。  

3. 判斷細節指引（部分重要 code 範例）：  
   - 「成團履約」(C-3-4)：包含未成團、臨時取消、擅自更改行程或餐點等執行面問題。  
   - 「駕駛接送-接送遲到」(C-3-2-2)：司機未準時出現、找不到人、電話聯絡不上等接送時間問題。  
   - 「人員服務-態度不佳」(C-3-1-1)：導遊、司機、接待等人員態度或專業講解能力不足。  
   - 「商品內容›行程流程›停留時間-每站」(C-1-2-6)：行程中每站停留時間安排與預期落差。  
   - 「商品內容›承諾與SLA›異動與通知義務」(C-1-8-5)：付款後行程變更時供應商主動通知的義務。  
   - 「客服營運›客服回應›AI鬼打牆」(C-5-3-2)：客服回覆內容制式化、答非所問、重複話術，或平台缺乏直接聯絡管道。  
   - 「客服營運›客服回應›已讀不回」(C-5-3-1)：客服有回應但等待時間過長或完全沒下文。  
   - 「客服營運›客服態度›態度推諉」(C-5-4-1)：客服態度冷淡、推諉、互踢皮球。  
   - 「客人理解›期待落差」(C-6-1)：旅客因自身期待與實際體驗不符的主觀落差。  

4. 歸類流程建議：  
   - 先判斷評論主體屬於哪個大類（商品內容、商品品質、供應商履約、使用體驗、客服營運、客人理解等）。  
   - 再根據評論具體內容，對照目錄中細項定義，選擇最符合的 L3 code。  
   - 若評論涉及多個問題，選擇最核心、最直接的問題歸類。  
   - 不要因為評論中有提及其他次要問題而誤判主體。  
   - 注意區分「商品內容」與「供應商履約」的界線：  
     - 商品內容偏向商品本身描述、定位、行程安排、費用資訊等。  
     - 供應商履約偏向執行過程中是否依約履行（如臨時取消、變更、服務態度、接送遲到等）。  

5. 輸出格式：  
   - 僅回傳符合的 L3 code（例如 C-3-4），不需其他文字說明。  
   - 若無法明確歸類，回空字串 ""。  

---

【判斷重點總結】  
- 嚴格且精準判斷評論的主體問題域。  
- 嚴格依據目錄定義，不跨域、不模糊。  
- 以評論中最核心、最直接的抱怨點為依據。  
- 明確區分商品內容與供應商履約。  
- 客服相關問題需區分客服回應內容、態度及流程設計缺失。  
- 客人理解類問題為旅客自身期待或誤解。  

---

【範例說明】  
- 食物口味差屬商品品質›餐飲品質›口味不佳 (C-2-2-1)  
- 司機未準時接送屬供應商履約›駕駛接送›接送遲到 (C-3-2-2)  
- 付款後行程變更未通知屬商品內容›承諾與SLA›異動與通知義務 (C-1-8-5)  
- 客服回覆制式化、答非所問屬客服營運›客服回應›AI鬼打牆 (C-5-3-2)  
- 旅客期待與實際落差屬客人理解›期待落差 (C-6-1)  
- 供應商擅自更改餐點屬供應商履約›成團履約 (C-3-4)
